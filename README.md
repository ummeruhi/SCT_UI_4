# 🏋️ Fitness App User Journey Prototype

## 📌 Task 04 – SkillCraft Technology Internship

This project demonstrates a **User Journey Design for a fictional Fitness Mobile Application**.  
The goal is to design a smooth onboarding experience that guides a user from the **welcome screen to initial fitness setup**.

The prototype simulates how a new user interacts with the application during the **first-time onboarding flow**.

## 2️⃣ Welcome Screen
Introduces the application and allows the user to begin the journey.

Main action:

## 3️⃣ Onboarding Screen 1

Purpose:

## 5️⃣ Account Access

# 🧠 UX Design Decisions

Several UX decisions were made to ensure a smooth experience:

### Simple Onboarding
Only two onboarding screens are used to avoid overwhelming users.

### Skip Option
Users can skip onboarding if they want to move faster.

### Progress Indicator
Shows users how far they are in the setup process.

### Minimal Profile Inputs
Only necessary details are collected to reduce friction.

### Multi-Goal Selection
Users can select multiple goals because real users often have more than one objective.

### Workout Preferences
Allows users to customize how they want to exercise.

### Notification Request at the End
Asking for permissions after showing value increases acceptance rates.

### Summary Confirmation
Helps users review their setup before entering the app.
